# v2 도메인 개념·색상 계약서

작성 기준일 2026-07-03.

이 문서는 v2 로직 어댑터를 만들기 전에 반드시 고정해야 하는 개념 계약이다. 목표는 `SeasonTag`, 퍼컬 시즌, 착용 계절, HEX, 색상이론, 의류 스키마가 서로 섞이지 않게 만드는 것이다.

## 1. 핵심 결론

v2에서는 "계절"이라는 단어를 단독으로 쓰지 않는다.

| 금지 표현 | v2 표현 | 의미 |
| --- | --- | --- |
| 계절 | 퍼컬 시즌 또는 보온 등급 또는 날씨 밴드 | 단독 사용 금지 |
| Season | `PersonalSeasonId` 또는 `WeatherBand` | 코드에서 수식어 없는 season 금지 |
| SeasonTag | v1 호환 입력값 | v2 저장·추천의 진실값으로 사용 금지 |
| 여름 옷 | `WarmthLevel='very-light' 또는 'light'` | 기온 적합성 |
| 여름 쿨 | `PersonalSeasonId='true-summer'` 계열 | 사람에게 어울리는 색채 유형 |

어댑터의 원칙은 다음 순서다.

1. v1 UI와 카탈로그 입력을 읽는다.
2. 색상 HEX, 색 이름, 카테고리, 종류, 소재, 패턴, 데님 여부를 정규화한다.
3. `SeasonTag`를 그대로 저장하지 않고 `WarmthLevel`을 파생한다.
4. 사용자 확정값이 있으면 자동 분석값보다 우선한다.
5. v2 추천 엔진에는 `PersonalSeasonId`, `WeatherBand`, `WarmthLevel`, `ColorSwatch`, `PatternType`, `MaterialType`, `isDenim`만 전달한다.

## 2. 개념 분리 지도

| 개념 | 질문 | 판정 대상 | 값 | 생성 방식 | 추천에서 쓰임 |
| --- | --- | --- | --- | --- | --- |
| 퍼컬 시즌 | 이 사람에게 어떤 색이 어울리는가 | 사람과 색 | 12개 `PersonalSeasonId` | 사진 ROI + 설문 4축 융합 | 퍼컬 적합도 38% 기본값 |
| HEX | 이 옷 또는 팔레트의 실제 색은 무엇인가 | 색상 샘플 | `#RRGGBB` | 팔레트 원본 또는 이미지 대표색 추출 | 퍼컬 거리, 조화, 중립색 판정 |
| 색 이름 | 사람이 읽는 색명은 무엇인가 | HEX의 라벨 | 화이트, 네이비, 데님 등 | `COLOR_META` 최근접 또는 텍스트 신호 | UI 표시, 규칙 보조 |
| 색상 조화 | 옷끼리 함께 있을 때 색이 어울리는가 | 코디 조합 | 조화 타입과 점수 | HSL hue 각도 + 명도·채도 균형 | 조화 축 28% 기본값 |
| 보온 등급 | 이 옷이 어느 기온대에 맞는가 | 의류 | `WarmthLevel` 5단계 | 카테고리·종류·소재·두께·커버리지에서 파생 | 날씨 축 22% 기본값 |
| 날씨 밴드 | 오늘 어느 기온 구간인가 | 외부 환경 | `WeatherBand` 8단계 | 위치 날씨 또는 수동 입력 | 보온 등급과 매핑 |
| 착용 계절 | 사람이 이해하기 쉬운 설명 | UI 보조 문구 | 봄·여름·가을·겨울 문구 | `WarmthLevel + WeatherBand`에서 표현 | 저장 필드 아님 |

## 3. 퍼스널컬러 시즌

퍼스널컬러 시즌은 사람과 색의 관계다. "라이트 서머라서 여름 옷을 입는다"가 아니라, "라이트 서머라서 특정 명도·채도·온도 범위의 색이 얼굴과 조화롭다"가 정확한 의미다.

### 3.3 퍼컬 적합도 계산 계약

의류 HEX가 퍼컬에 맞는지 보는 방식은 다음이다.

1. 의류의 대표 HEX와 dominant HEX 상위 3개를 읽는다.
2. 각 HEX를 CIELAB으로 변환한다.
3. 사용자 Top1 시즌 팔레트의 각 HEX와 Delta E 2000 거리를 계산한다.
4. 가장 가까운 팔레트 색의 거리로 점수를 만든다.
5. 진단 신뢰도가 낮으면 Top2 팔레트를 최대 35% 섞는다.
6. 회피색에 가까우면 감점한다.
7. 중립색·데님은 코디 유틸리티로 소폭 가산할 수 있다.

Delta E 해석은 다음 기준을 쓴다.

| 거리 | 의미 | v2 점수 해석 |
| ---: | --- | --- |
| 0~5 | 거의 같은 계열 | 매우 적합 |
| 5~12 | 눈에 보이나 자연스러운 차이 | 적합 |
| 12~22 | 같은 계열로 볼 수 있으나 차이 큼 | 보통 |
| 22~35 | 팔레트와 거리가 있음 | 주의 |
| 35 초과 | 팔레트 밖 | 낮은 적합 |

## 4. 날씨와 보온 등급

날씨는 외부 환경이고, 보온 등급은 옷의 속성이다. 둘을 직접 매핑해야 하며, `SeasonTag` 문자열로 하드컷하지 않는다.

### 4.1 WeatherBand

| v1 UI 라벨 | v2 내부값 | 기온 | 의미 |
| --- | --- | --- | --- |
| 4도 이하 | `freezing` | <= 4도 | 한겨울 |
| 5~8도 | `cold` | 5~8도 | 추움 |
| 9~11도 | `chilly` | 9~11도 | 쌀쌀함 |
| 12~16도 | `cool` | 12~16도 | 선선함 |
| 17~19도 | `mild` | 17~19도 | 온화함 |
| 20~22도 | `warm` | 20~22도 | 따뜻함 |
| 23~27도 | `hot` | 23~27도 | 더움 |
| 28도 이상 | `very-hot` | >= 28도 | 한여름 |

### 4.2 WarmthLevel

| 값 | 의미 | 대표 의류 | 이상 날씨 밴드 | 보조 허용 |
| --- | --- | --- | --- | --- |
| `very-light` | 열 배출 중심 | 민소매, 얇은 반팔, 반바지, 샌들 | `hot`, `very-hot` | `warm` |
| `light` | 얇고 가벼움 | 반팔, 얇은 셔츠, 얇은 스커트 | `warm`, `hot` | `mild`, `very-hot` |
| `mid` | 기본 두께 | 긴팔티, 셔츠, 슬랙스, 일반 데님, 스니커즈 | `mild`, `warm`, `cool` | `chilly`, `hot` |
| `warm` | 보온 레이어 | 니트, 가디건, 재킷, 트렌치, 부츠 | `chilly`, `cool`, `cold` | `mild`, `freezing` |
| `heavy` | 방한 | 패딩, 울 코트, 두꺼운 니트, 머플러 | `freezing`, `cold` | `chilly` |

### 4.3 WarmthLevel 파생 순서

어댑터는 다음 순서로 `WarmthLevel`을 만든다.

1. 사용자가 확정한 보온 등급이 있으면 그것을 사용한다.
2. 의류 종류와 카테고리에서 1차 후보를 만든다.
3. 소재와 두께 관측값으로 보정한다.
4. 노출·소매·길이 같은 구조 관측값이 있으면 보정한다.
5. v1 `SeasonTag`는 마지막 호환 폴백으로만 사용하고 `confidenceNotes`에 경고를 남긴다.

`SeasonTag` 폴백은 다음처럼 약하게만 해석한다.

| v1 SeasonTag | v2 폴백 | 신뢰도 |
| --- | --- | --- |
| 여름 | `light` | 낮음 |
| 봄/가을 | `mid` | 낮음 |
| 겨울 | `warm` | 낮음 |
| 사계절 | `mid` | 매우 낮음 |

## 5. 색상이론 계약

v2 추천은 색을 두 방식으로 본다.

첫째, 사람에게 맞는 색인지 본다. 이때는 퍼컬 팔레트와 의류 HEX의 Delta E 거리를 쓴다.

둘째, 옷끼리 어울리는지 본다. 이때는 HSL hue 각도, 명도 균형, 채도 균형을 쓴다.

### 5.1 조화 타입

| Hue 각도 차이 | 타입 | 의미 | 기본 점수 |
| ---: | --- | --- | ---: |
| 0~15도 | `monochromatic` | 톤온톤 | 80 |
| 16~45도 | `analogous` | 유사색 | 82 |
| 46~90도 | `tension` | 긴장 배색 | 55 |
| 91~135도 | `triadic` | 삼각 배색 | 76 |
| 136~180도 | `complementary` | 보색 대비 | 88 |
| 무채색 포함 | `neutral` | 중립 안정 | 85 기준 |

### 5.2 조화 점수 성분

| 성분 | 비중 | 목적 |
| --- | ---: | --- |
| hue 관계 | 50% | 색상환상의 관계 |
| 명도 균형 | 30% | 상하의 분리감과 가독성 |
| 채도 균형 | 20% | 과한 포인트 충돌 방지 |

시즌 대비 선호도도 반영한다. 고대비 시즌은 보색을 더 잘 받아들이고, 저대비 시즌은 유사색·톤온톤이 더 안전하다.

## 6. 의류 스키마 계약

v2 의류 스키마는 추천에 실제로 쓰이는 필드를 중심으로 둔다. 아래는 어댑터가 목표로 해야 하는 정규화 형태다.

```ts
  denimWash?: 'light' | 'mid' | 'dark' | 'black';
  availability: 'owned' | 'laundry' | 'stored' | 'excluded';
  analysis: {
    status: 'manual' | 'draft' | 'confirmed';
    confidence: number;
    modelRefs: string[];
    observations: string[];
    warnings: string[];
    confirmedAt?: string;
  };
  createdAt: string;
  updatedAt: string;
}
```

### 6.1 필드별 출처와 파생 규칙

| 필드 | 1차 출처 | 2차 출처 | 금지 |
| --- | --- | --- | --- |
| representative HEX | 이미지 대표색 또는 카탈로그 메타 | 색 이름 매핑 | 색 이름을 HEX처럼 저장 금지 |
| dominant HEX | 이미지 색상 비율 상위 3개 | 대표 HEX 복제 | 비율 없는 다색을 같은 가중치로 단정 금지 |
| material | 사용자 확정값 | 상품명·종류·VLM 관측 | `pants -> cotton` 고정 매핑 금지 |
| pattern | 사용자 확정값 | 상품명·이미지 관측 | 이미지 없는 상태에서 전부 solid 단정 금지 |
| isDenim | 종류·상품명·소재·색의 교차 검증 | HEX 데님 근접 | `isDenim=true`와 `material='cotton'` 동시 저장 금지 |
| warmthLevel | 사용자 확정값 또는 의류 속성 파생 | v1 SeasonTag 폴백 | 색 밝기로 보온 등급 단정 금지 |
| analysis.status | 사용자 확인 여부 | 없음 | draft를 confirmed처럼 추천 근거에 쓰기 금지 |

### 6.2 데님 정합성

데님은 색이 아니라 소재·직물·의류 종류가 결합된 속성이다. 따라서 파란색이라고 모두 데님이 아니고, 검은 청바지도 데님이다.

데님 판정 우선순위는 다음이다.

1. 사용자 확정 `material='denim'` 또는 종류 `청바지`.
2. 상품명에 `청`, `데님`, `denim`, `jean` 계열이 있음.
3. 자동 관측에서 데님 텍스처 또는 데님 소재가 나옴.
4. HEX가 데님 팔레트에 가깝다는 사실은 보조 신호로만 사용한다.

정합성 규칙은 다음이다.

| 조건 | 저장 결과 |
| --- | --- |
| `isDenim=true` | `material='denim'`이어야 함 |
| `material='denim'` | `isDenim=true`이어야 함 |
| `denimWash` 존재 | `isDenim=true`이어야 함 |
| 검은 청바지 | `isDenim=true`, `denimWash='black'` 허용 |
