# v2 재구축 계획

작성일: 2026-07-02.

## 목표

현재 저장소의 v1을 직접 수리하지 않고, 이 폴더 안에서 `v2`를 새 제품처럼 재구축한다. v1은 검증된 도메인 지식과 UI 참고 자료로만 사용하고, 의류 데이터 스키마·수집 파이프라인·추천 로직·프로젝트 구조는 새로 정리한다.

## 핵심 가정

- `v2/`는 새 앱과 문서의 기준 폴더다.
- `v2_md/`는 기존 문서 보관 폴더로 보고, 새 작업의 기준은 `v2/`로 통일한다.
- 골든 패스는 외부 네트워크 없이 돌아야 한다.
- 사진 등록과 URL 등록은 1차 시연 경로 밖의 부가 기능이다.
- 퍼스널컬러 시즌과 착용 계절은 타입·문구·로직에서 분리한다.

## 단계

1. P0 문서·계약 정리.
   - `v2/docs/domain-logic-inventory.md`에 v1에서 승계할 로직 정보를 정리한다.
   - `v2/docs/v2-schema-contract.md`에 새 의류·퍼컬·추천 스키마를 고정한다.
   - `v2/checklist.md`, `v2/context-notes.md`를 날짜별 누적 문서로 유지한다.

2. P0 스캐폴드.
   - React 19 + TypeScript + Vite 앱을 `v2/app` 또는 `v2` 루트 패키지로 구성한다.
   - FastAPI 서버는 `v2/server`에 새로 둔다.
   - 데이터 파이프라인은 `v2/pipeline`, 설정은 `v2/config`에 둔다.

3. P1 골든 패스.
   - 카탈로그 기반 프리셋 옷장을 먼저 만든다.
   - 겹치지 않는 추천 3개가 항상 성립하도록 검증 스크립트를 둔다.
   - 추천 결과를 데일리룩으로 저장하고 PNG로 내보낸다.

4. P2 부가 수집 경로.
   - URL 수집과 사진 등록을 같은 의류 분석 스키마로 연결한다.
   - 자동 인식은 초안 작성기로만 쓰고, 사용자 확정값을 최종 권한으로 둔다.

5. P3 논문 트랙.
   - v1 RF, 임베딩 분류 헤드, VLM 제로샷을 동일 라벨 516장으로 비교한다.
   - 서비스 구현과 논문 실험을 분리해 일정 리스크를 줄인다.

## 1차 완료 기준

- `npm test` 통과.
- `npm run build` 통과.
- 프리셋 옷장에서 겹치지 않는 추천 3개가 나온다.
- 추천 근거와 축별 점수가 화면에 표시된다.
- 데일리룩 이미지 저장이 된다.
- 골든 패스는 네트워크 없이 완주된다.
