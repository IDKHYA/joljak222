# v2 재구축 기억 메모

작성일: 2026-07-02.

## 반드시 기억할 것

- 이 작업은 v1 수리가 아니라 `v2/` 새 폴더에서 크게 다시 재코딩하는 작업이다.
- UI 구성과 화면 흐름은 v1 원본을 계승한다. 홈, 퍼컬 측정, 설문, 결과, 옷장, 추천, 저장 코디, 데일리룩이 v2 앱에 함께 있어야 한다.
- 첫 v2 골든 패스 대시보드는 최종 UI가 아니라 smoke prototype이다. 실제 진입점으로 키우지 않는다.
- 로직도 새로 정리한다. v1의 구현 방식을 성역으로 두지 않는다.
- 그래도 v1의 퍼스널컬러 팔레트, 설문 축, 추천 4축, 이미지 저장 방식, UI 흐름은 검증된 자산이다.
- 먼저 스키마와 로직 계약을 문서로 고정한다.
- 이후 테스트를 먼저 쓰고 최소 구현으로 통과시킨다.

## 문서 기준

- 실행 명세: `리버스frd.md`.
- 설계 배경: `설계자_관점.md`.
- v2 계획: `v2/plan.md`.
- 진행 체크: `v2/checklist.md`.
- 의사결정 기록: `v2/context-notes.md`.
- 로직 인벤토리: `v2/docs/domain-logic-inventory.md`.
- 스키마 계약: `v2/docs/v2-schema-contract.md`.

## 다음 세션 시작 순서

1. `v2/context-notes.md`를 먼저 읽는다.
2. `v2/checklist.md`에서 미완료 항목을 확인한다.
3. 코드 작업 전 `v2/docs/domain-logic-inventory.md`와 `v2/docs/v2-schema-contract.md`를 확인한다.
4. 원본 UI 이식 작업은 prototype 대시보드를 먼저 분리하고, v1 원본 화면 흐름을 v2 진입점으로 연결한다.
5. 카탈로그 작업은 v2 안에 이미지, 메타데이터, 매니페스트, 검증 테스트를 함께 둔다.
6. 새 소스 파일 첫 줄에는 한국어 역할 주석을 넣는다.
7. 코드 변경 후 테스트 또는 빌드를 실행한다.
