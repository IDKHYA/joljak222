## 반드시 기억할 것

- 이 작업은 v1 수리가 아니라 `v2/` 새 폴더에서 크게 다시 재코딩하는 작업이다.
- 로직도 새로 정리한다. v1의 구현 방식을 성역으로 두지 않는다.
- 그래도 v1의 퍼스널컬러 팔레트, 설문 축, 추천 4축, 이미지 저장 방식, UI 흐름은 검증된 자산이다.
- 먼저 스키마와 로직 계약을 문서로 고정한다.
- 이후 테스트를 먼저 쓰고 최소 구현으로 통과시킨다.
