## 2. 퍼스널컬러 진단 파이프라인

### 2-1. 개요

사진 분석과 설문 두 축을 **동적 가중치로 융합**해 12계절 중 하나를 최종 결과로 낸다. 사진이 잘 찍힐수록 사진 비중이 올라가고, 조명이 나쁠수록 설문이 더 크게 반영된다.

핵심 로직 파일: `src/services/personalColorEngine.ts` (448줄, 외부 AI API 없음, 순수 계산)

### 2-2. 12계절 분류 체계

```
봄 계열 (웜)
  - light-spring    라이트 스프링    따뜻 + 밝음 + 중간 선명도
  - true-spring     트루 스프링      따뜻 + 중간 명도 + 선명
  - bright-spring   브라이트 스프링  따뜻 + 매우 선명 + 고대비

여름 계열 (쿨)
  - light-summer    라이트 서머      차가움 + 매우 밝음 + 소프트
  - true-summer     트루 서머        차가움 + 중간 명도 + 뮤트
  - soft-summer     소프트 서머      차가움 + 어두움 + 매우 뮤트

가을 계열 (웜)
  - soft-autumn     소프트 오텀      따뜻 + 중간 명도 + 뮤트
  - true-autumn     트루 오텀        따뜻 + 어두움 + 약간 선명
  - dark-autumn     다크 오텀        따뜻 + 매우 어두움 + 약간 뮤트

겨울 계열 (쿨)
  - dark-winter     다크 윈터        차가움 + 어두움 + 고대비
  - true-winter     트루 윈터        차가움 + 중간 명도 + 매우 선명 + 극고대비
  - bright-winter   브라이트 윈터    차가움 + 선명 + 최고대비
```

각 시즌은 4축 `traits` 벡터 (temperature, lightness, clarity, contrast, 각각 -1~1) 로 수치화되어 있다. (`personalColorWorkbook.ts`)

### 2-3. 사진 분석 — `photoAnalysis.ts`

**MediaPipe 얼굴 랜드마크** → **15개 ROI 박스 생성** → **픽셀 샘플링** → **대표색 추출** → **품질 점수**

**ROI (관심 영역) 목록.**

| 키 | 설명 |
|----|------|
| skinLeft / skinRight | 좌우 볼 (가장 크게 반영) |
| forehead | 이마 중심 |
| noseLeft / noseRight | 코 좌우 |
| underEyeLeft / underEyeRight | 눈 아래 |
| jawLeft / jawRight | 턱선 |
| eyesLeft / eyesRight | 홍채 |
| eyebrowLeft / eyebrowRight | 눈썹 |
| lips | 입술 |
| hair | 헤어라인 |

**피부 대표색 가중치.** 볼 40% + 이마 14% + 코 16% + 눈밑 20% + 턱선 10%

**샘플링 방식.**
- 일반 ROI: 루미넌스 기준 상하위 픽셀 트리밍 후 평균
- 피부 ROI: 채도 필터 + Lab medoid (이상치에 강한 중앙값 선택)
- 입술 ROI: 붉은/분홍 색조 픽셀 우선 선별, 부족하면 전체 픽셀 fallback
- 헤어라인: 배경 섞임 방지를 위해 눈썹색 40% 혼합

**조명 보정 (3단계 fallback).**
1. 흰 종이 기준 — 사용자가 흰 종이를 들고 촬영한 경우
2. 배경 중립 픽셀 — 얼굴 주변 배경의 무채색 픽셀 수집
3. 코너 fallback — 화면 네 모서리 12% 패치의 평균

**사진 품질 점수 (0~1).**
```
overall = 0.12 + 노출점수 × 0.24 + 대칭점수 × 0.18 + 구분도 × 0.18 + 얼굴크기 × 0.12 + 배경점수 × 0.16
          (하한 0.35, 상한 0.98)
```

출력: `PhotoAnalysisPayload` = `{extractedColors: {skin, hair, eyes, lips}, photoQuality, measurementDetails}`

### 2-4. 색상 특징 벡터 — `measureColorFeatures()`

사진에서 추출한 skin/hair/eyes/lips를 5개 특징으로 변환한다.

**temperature (색온도) — Lab b* 축 기반.**
```
temperature = labTemperatureIndex(skin)×0.45 + labTemperatureIndex(lips)×0.25
            + labTemperatureIndex(hair)×0.15 + labTemperatureIndex(eyes)×0.15
```
기존 RGB 채널 차이 방식은 조명 색온도에 흔들림. Lab b*는 노랑(웜)↔파랑(쿨)을 perceptually uniform space에서 표현해 조명 변화에 덜 민감하다.

**contrast (대비) — 동아시아 과분류 보정.**
```
facialContrast = max(luminances) - min(luminances) for [skin, eyes, lips]
hairContrast = |luminance(skin) - luminance(hair)|
contrast = facialContrast×0.78 + hairContrast×0.22
```
머리카락 비중을 22%로 낮춤. 동아시아 사용자의 검은 머리만으로 겨울 고대비가 되는 과분류 문제 완화.

### 2-5. 설문 분석 — `personalColorEngine.ts`

**5문항** (이전 8문항에서 white_clothing, sun_reaction, muted_colors 3개 제거. 신호 강도가 낮거나 다른 문항과 중복돼 노이즈 유발).

| 문항 | 핵심 신호 |
|------|----------|
| 손목 혈관 색 (올리브/블루) | temperature ±0.55 |
| 액세서리 (골드/실버) | temperature ±0.65 |
| 선명한 컬러 착용 반응 | clarity ±0.7, contrast ±0.3 |
| 대비 스타일 선호 | contrast ±0.7, clarity ±0.15 |
| 색의 깊이 선호 | lightness ±0.7 |

`calculateQuestionnaireScores()`: 5문항 가중치 합산 → 각 축 최대값으로 나눠 -1~1 정규화

### 2-6. 사진 시즌 점수 계산

각 12시즌마다 두 가지 점수를 계산해 합친다.

**팔레트 점수 (paletteScore). — CIE76 기준**
```
= scorePaletteMatch(skin) × 0.45
+ scorePaletteMatch(hair) × 0.2
+ scorePaletteMatch(eyes) × 0.15
+ scorePaletteMatch(lips) × 0.2
```

**traits 유사도 (traitScore).**
```
= closeness(temperature) × 0.30
+ closeness(lightness) × 0.16
+ closeness(clarity) × 0.34   ← 선명도 비중 가장 높음 (브라이트/소프트/다크/라이트 분류 핵심)
+ closeness(contrast) × 0.20
```

**도메인 보정 규칙.**
- 저채도 얼굴(mutedScore 높음)이 겨울 고선명 시즌으로 과분류되는 것을 막기 위해 겨울 계열에 최대 0.16 페널티
- soft-summer, soft-autumn은 뮤트 신호가 강할 때 0.1 보너스
- 고대비 시즌인데 사진 대비 낮으면 0.12 페널티
- 고선명 시즌인데 사진 선명도 낮으면 0.14 페널티

**최종 사진 시즌 점수 = paletteScore × 0.42 + traitScore × 0.58**

### 2-7. 퓨전 (사진 + 설문) — `fuseResults()`

**동적 가중치.**
```typescript
photoWeight = clamp(0.22 + photoQuality × 0.14, 0.22, 0.36)
questionnaireWeight = 1 - photoWeight
```

사진 품질이 0이면 사진 22% : 설문 78%, 사진 품질이 1이면 사진 36% : 설문 64%.

**신뢰도(confidence).**
```
0.42 + top1_score × 0.28 + gap × 1.35 + photoQuality × 0.12 + consistencyBonus
```
- consistencyBonus: 사진과 설문 Top1 시즌이 같으면 0.08, 같은 계열이면 0.03
- gap < 0.06이면 경계 시즌 안내 표시

출력: `FinalResult` = 시즌 Top1/Top2, 팔레트(24색), 신뢰도, 근거, 추천 특성, 측정 상세

### 2-8. 참조 데이터 — `personalColorWorkbook.ts`

12계절 × 24색 팔레트 (총 288 HEX). 엑셀 원본에서 변환된 수치.

- 각 시즌마다 averageRgb, averageLightness, averageSaturation, averageTemperature, averageContrast 통계 보유
- 모듈 로딩 시 전체 팔레트를 Lab 좌표로 캐싱 (`SEASON_PALETTE_LABS`)

**피해야 할 색 — `seasonContent.ts`.**

12계절 × 5색 = 60 HEX. 추천 엔진에서 의류 대표색이 이 색과 가까우면 감점(avoidPenalty).

## 3. 옷 이미지 처리 파이프라인

### 3-1. 개요

옷 사진을 업로드하면 두 단계로 처리된다.

```
1단계: /api/background/remove  — rembg(u2netp) 일반 배경 제거
2단계: /api/clothing/extract   — SegFormer 정밀 의류 추출 + 계절/재질 예측
```

두 엔드포인트 모두 FastAPI (`background_remove_api.py`) 에서 처리. 두 모델 모두 lazy load (첫 요청 시 로드, 이후 전역 캐싱).

### 3-2. 일반 배경 제거 — `/api/background/remove`

- 모델: `rembg u2netp` (일반 범용 배경 제거)
- 처리: 업로드 → resize (최대 1600px) → rembg 배경 제거 → `harden_alpha()` 반투명 제거 → PNG data URL
- 반환: `{imageDataUrl, width, height, bbox, colors, model, version, processedAt}`

`harden_alpha()`: alpha < 56은 완전 투명, ≥ 56은 완전 불투명으로 이진화. 가장자리 1px는 부드럽게 남김.

### 3-3. 정밀 의류 추출 — `/api/clothing/extract`

**SegFormer (`sayeed99/segformer-b3-fashion`).**
- 픽셀별 의류 카테고리를 예측하는 semantic segmentation 모델 (HuggingFace)
- 39개 라벨 (shirt, pants, coat, shoes 등)

**처리 흐름.**
```
업로드 이미지
  → resize (최대 640px, 추론 속도용)
  → SegFormer → 픽셀별 label map (pred, numpy array)
  → target_ids(targetPart)로 부위 mask 추출
  → dress/jumpsuit는 상하의 요청에 자동 포함
  → clean_boolean_mask() — morphology close/open + fill_holes + 상위 3개 연결 성분만 유지
  → rgba_from_boolean_mask() → PNG data URL
  → predict_season(pred, mask, image, targetPart)  ← season_predictor.py
  → 반환
```

**부위별 label 집합.**
| targetPart | SegFormer label ids |
|------------|---------------------|
| upper | 1(shirt), 2(top), 3(sweater), 6(vest), 28(hood), 29(collar), 32(sleeve), 34(neckline) |
| lower | 7(pants), 8(shorts), 9(skirt) |
| outer | 4(cardigan), 5(jacket), 10(coat), 13(cape) + hood/collar/sleeve/neckline |
| shoes | 24(shoe) |
| accessory | 14~27 (glasses, hat, scarf 등) |

### 3-4. 계절/재질 예측 — `season_predictor.py`

이 모듈이 **졸업 프로젝트의 핵심 독자 기여**다.

**입력.** `predict(pred, mask, pil_img, target_part)` — SegFormer 예측 결과 전체 + 의류 마스크 + 원본 이미지 + 부위

**파이프라인.**

```
① 라벨 추출
   infer_labels_from_pred()
   → fine_labels: 주요 의류 라벨 (sweater, pants 등)
   → detail_labels: 디테일 라벨 (hood, zipper, pocket 등)

② 대표색 추출
   get_dominant_colors()
   → KMeans + silhouette score로 최적 k 자동 선택 (2~5)
   → 마스크 영역 픽셀만 클러스터링

③ 룰 기반 예측 (fallback 겸 피처)
   get_rule_season()
   → 라벨별 계절 점수 테이블로 봄/가을, 여름, 겨울 점수 합산
   → 소매 비율 (sleeve_ratio): 짧으면 여름, 길면 봄/가을+겨울
   → 노출 비율 (exposure): 작으면 여름, 크면 겨울
   → 색상 밝기: 밝으면 여름 가중, 어두우면 겨울 가중

④ 97개 피처 추출
   _build_feature_row()
   → label_pixel_ratios: 34개 라벨별 픽셀 비율
   → bbox: 위치, 크기, 면적 비율
   → shape: 높이/너비 비율, 채움 비율
   → color_stats: 밝기 평균/분산, 채도, warm/cool/neutral 비율 등 13개
   → quality: 전체 이미지 대비 의류 픽셀 비율 등
   → fine_label 원핫, detail_label 원핫
   → 색상 상위 5개 RGB + 비율
   → 룰 점수 3개 + 룰 신뢰도 + 룰 예측 계절 원핫

⑤ ML 예측
   RandomForest (300 estimators, balanced class weights)
   학습: 516개 직접 라벨링 이미지
   피처: 97개 컬럼 (feature_columns.json)
   클래스: ['겨울', '봄/가을', '여름']
   테스트 정확도: 89.74% (룰 기반 베이스라인 19.2% 대비 약 4.7배)

⑥ 재질 예측
   infer_material()
   fine_label → material 매핑 테이블
   (sweater→knit, coat→wool, jacket→nylon, pants→cotton 등)
```

**출력.**
```python
{
  "predictedSeason": "봄/가을" | "여름" | "겨울" | "미분류",
  "seasonConfidence": 0.0 ~ 1.0,
  "seasonProbabilities": {"봄/가을": 0.x, "여름": 0.x, "겨울": 0.x},
  "predictedMaterial": "cotton" | "knit" | "wool" | ...
}
```

ML 모델 파일이 없으면 룰 기반 결과로 fallback.

### 3-5. 대표색 추출 — `extract_dominant_colors()`

누끼 결과의 불투명 픽셀 (alpha ≥ 96) 만 사용. 너무 밝은 배경 잔여 픽셀 제외. 24단계 버킷으로 양자화 후 최대 5개 반환.

## 6. 추천 알고리즘

핵심 로직 파일: `src/services/recommendationEngine.ts` (2026-05-22 App.tsx에서 분리, 단위 테스트 `recommendationEngine.test.ts` 동반). 아래 함수들(`scoreItemForPersonalColor`, `isWeatherEligible`, `getWeatherScore`, `calculateHarmonyScore`, `buildRecommendations` 등)이 모두 이 모듈에 모여 있다.

### 6-1. 전체 흐름

```
1. 전체 의류 → scoreItemForPersonalColor() → ScoredClothingItem[]
2. 추천제외/세탁중 제거 + 날씨 필터 → 후보 의류
3. 카테고리별 분류 → tops, bottoms, outerwear
   (신발은 2026-05-15부터 조합 제외 — "신발은 어디든 있다" 피드백)
4. 조합 생성 → top × bottom × (outer|없음)
5. diversifyRecommendations() — maxPerItem=3, 동일 아이템 독점 방지
6. 점수 내림차순 정렬 → 상위 60개
```

### 6-2. 의류 개별 퍼스널컬러 점수 — `scoreItemForPersonalColor()` (CIEDE2000)

```
dominantColors 상위 3색 각각 scoreSingleHex() → ratio 가중 평균

scoreSingleHex(hex):
  itemLab = HEX → RGB → Lab
  paletteDistance = min(CIEDE2000(itemLab, paletteLab))  ← 24색과 최소 거리
  paletteScore = max(0, 100 - paletteDistance × 4.5)

  avoidDistance = min(CIEDE2000(itemLab, avoidLab))      ← 피해야 할 5색과 최소 거리
  avoidPenalty = avoidDistance < 10 ? 22 : avoidDistance < 16 ? 10 : 0

  utilityBonus = isNeutral || isDenim ? 8 : 0
  score = clamp(paletteScore + utilityBonus - avoidPenalty, 0, 100)
```

등급: BEST ≥ 88 / GOOD ≥ 74 / OK ≥ 58 / CHECK < 58

**CIEDE2000 채택 이유.** 파란 계열 색상에서 CIE76 상관계수 0.176 vs CIEDE2000 0.289 (PubMed 기준). 쿨 퍼스널컬러 팔레트에서 정확도가 더 높음.

### 6-3. 날씨 필터 — `isWeatherEligible()`

```typescript
// 9도 이하 → 여름옷 제외
if (item.seasonTag === '여름' && bandIndex <= 2) return false;
// 20도 이상 → 겨울옷 제외
if (item.seasonTag === '겨울' && bandIndex >= 5) return false;
```

### 6-4. 날씨 점수 — `getWeatherScore()`

```
보유중 기준 기본 60점
+ 현재 날씨 구간 허용 키워드에 type이 포함되면 +28
+ seasonTag === '사계절' ? +8
+ 28도 이상이고 seasonTag === '겨울' ? -30
+ 4도 이하/5~8도이고 seasonTag === '여름' ? -25
세탁중: 최대 20점 / 보관중: 최대 45점 / 추천제외: 0점
```

### 6-5. 색상 조화 점수 — Itten 색상 이론 5타입

hue 각도 기반으로 분류 (기존 이진 84/72 방식 → Lab ΔE → hue 5타입으로 3단계 발전).

```
monochromatic (단색):  hue 차 ≤ 15도
analogous (유사색):    hue 차 ≤ 45도
tension (포인트 배색): hue 차 ≤ 90도
triadic (삼각):        hue 차 ≤ 135도
complementary (보색):  hue 차 > 135도
중성색(isNeutral):     기본 85점
```

각 타입의 기본 점수는 `HARMONY_BASE_SCORES`로 정의된다. monochromatic 80, analogous 82, tension 55, triadic 76, complementary 88, 중성색 85. 여기에 두 가지 보정이 더해진다.

- **시즌 대비 보정.** 보색 조합은 대비 선호 시즌(traits.contrast > 0.5)에서 +6, 저대비 시즌(< -0.2)에서 -10. 유사색 조합은 저대비 시즌에서 +6.
- **패턴 페널티 (`calculatePatternPenalty`).** 그래픽 + 다른 패턴 조합 -22, 같은 패턴 중복 -14, 그 외 패턴 충돌 -8. 무지(solid)는 페널티 없음.

### 6-6. 코디 종합 점수 (2026-05-15 재조정)

```
score = personalScore × 0.38    ← 퍼스널컬러 적합도 (기존 0.42)
      + weatherScore  × 0.22    ← 날씨 적합도 (기존 0.28)
      + harmonyScore  × 0.28    ← 색상 조화 (기존 0.20, 상향)
      + stabilityScore × 0.12  ← 보유 안정성 (기존 0.10)
```

stabilityScore: 전부 보유중 → 92, 아니면 68.

- 색상 조합(hue bucket) 기반 pill 필터 (8개 버킷: 레드/오렌지/옐로/그린/시안/블루/퍼플/핑크/뉴트럴)

## 11. 중요 구분 — 혼동하면 안 되는 것들

### 퍼스널컬러 계절 ≠ 옷의 날씨 계절

| 구분 | 예시 | 어디에 쓰이나 |
|------|------|--------------|
| **퍼스널컬러 계절** | light-spring, soft-autumn, true-winter | FinalResult.seasonTop1Id, 팔레트 비교 |
| **옷의 날씨 계절** | '봄/가을', '여름', '겨울', '사계절' | ClothingItem.seasonTag, 날씨 필터 |

### CatalogItem ≠ ClothingItem

`CatalogItem`은 쇼핑 목록 개념. 사용자가 '내 옷장에 추가'를 누를 때 비로소 `fromCatalog()`로 `ClothingItem`이 복사 생성된다.

### scorePaletteMatch vs scoreItemForPersonalColor 색상 거리 기준

- **scorePaletteMatch** (진단 팔레트 매칭): **CIE76** 유지 — 진단 단계에서 속도가 우선
- **scoreItemForPersonalColor** (추천 점수): **CIEDE2000** — 정확도가 우선, 파란 계열 쿨 팔레트 상관계수 0.289

### personalColorEngine.ts 구 명칭 주의

과거 파일명이 `geminiService.ts`였으나 2026-05-15에 `personalColorEngine.ts`로 리네임. Gemini API 미사용. 순수 룰 엔진.
