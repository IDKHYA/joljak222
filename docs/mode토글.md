# mode 토글 (데일리 / 오피스 / 데이트) — 제거 기록과 향후 계획

작성일: 2026-05-26
대상: `src/features/recommendation/AnchorOutfitFinder.tsx`, `src/services/recommendationEngine.ts`

---

## 1. 결론

`AnchorOutfitFinder`의 추천 방식 토글(데일리/오피스/데이트)을 **지금은 제거**했다. 기능은 **나중에 제대로 추가**한다. 이 문서는 "왜 뺐고, 다시 넣을 땐 이렇게 한다"를 남기는 기록이다.

---

## 2. 왜 제거했나

토글은 화면상 동작하는 것처럼 보였지만 **결과를 전혀 바꾸지 못했다.**

- 엔진에서 `mode`는 `scoreOutfit()`의 **제목 문자열**에만 쓰인다. 예: `"선명한 대비 오피스 코디"`.
- `buildAnchoredRecommendations`는 날씨 축을 `'상관없음'`으로 고정한다. `mode`는 점수(퍼스널컬러·날씨·조화·안정성)에도, 후보 필터링에도 관여하지 않는다.
- 결과적으로 데일리·오피스·데이트는 **조합도 점수도 완전히 동일**하고, 바뀌는 건 카드 제목의 단어뿐이었다.
- 화려한 3버튼 토글(아이콘·설명 포함)로 노출돼 있어서, 눌러도 안 바뀌는 게 오히려 "고장난 것"처럼 보였다.

즉 "있는데 안 먹는" 상태였다. 어설프게 두는 것보다 빼고, 제대로 만들 때 다시 넣기로 했다.

---

## 3. 지금 적용한 변경

- `AnchorOutfitFinder.tsx`에서 `MODE_OPTIONS`, `mode` 상태, "추천 방식" 토글 UI, 관련 아이콘 import를 제거했다.
- `buildAnchoredRecommendations` 호출에서 `mode` 인자를 빼 기본값(`'데일리'`)을 쓴다. 이에 따라 `results` 메모의 의존성에서도 `mode`가 빠져, mode 변경 시 발생하던 불필요한 재계산도 사라졌다.
- **CSS(`.anchor-finder-mode` 계열)는 일부러 남겨 뒀다.** `.anchor-finder-source`와 선택자를 공유하고, 나중에 토글을 다시 넣을 때 그대로 재사용할 수 있기 때문이다.

엔진의 `mode` 파라미터와 `RecommendationMode` 타입은 그대로 둔다. 메인 추천 대시보드(`RecommendationDashboard`)도 같은 `mode`를 쓰므로 건드리지 않는다.

---

## 4. 앞으로 이렇게 한다 (재추가 계획)

핵심 방향. **`mode`가 제목이 아니라 점수에 실제로 반영되게 만든다.**

### 4.1 엔진에 mode 가중치 추가

`scoreOutfit()` 혹은 조화 점수 계산부에 `modeAffinityBonus(item, mode)` 같은 작은 보정을 더한다. 팔레트·조화 점수를 뒤집지 않도록 가중은 작게(대략 ±5~10점) 둔다.

| mode | 가산(선호) | 감점(비선호) |
| --- | --- | --- |
| 데일리 | 보정 없음(기준선) | — |
| 오피스(출근) | 셔츠·블라우스·슬랙스·블레이저·코트·니트, 무채색·저채도 | 후드·트레이닝·반바지·그래픽 프린트 |
| 데이트 | 원피스·스커트·니트, 부드럽고 따뜻한 색, 중채도 | 지나치게 딱딱한 정장 느낌 |
| 발표 | 오피스와 유사하되 격식 가산을 더 강하게 | 캐주얼 전반 |

### 4.2 데이터는 우선 type 키워드로 시작

지금 의류에는 격식(formality)·무드 메타데이터가 없다. v1 재추가 때는 `item.type` 문자열의 키워드 매칭으로 격식을 추정한다(예: `셔츠`·`슬랙스`·`블레이저` 포함 → 격식 ↑). 정확도를 더 높이려면 다음 단계로.

- `formality` 같은 필드를 의류 메타데이터에 추가.
- 또는 Tier 3 LLM 비전으로 옷의 격식/무드를 태깅(점수 계산은 엔진이 유지, LLM은 데이터 입력만).

### 4.3 UI 복구

`MODE_OPTIONS` 토글을 `AnchorOutfitFinder`에 다시 넣는다. CSS는 이미 남아 있어 마크업만 복구하면 된다. `results` 메모 의존성에 `mode`를 다시 추가한다.

### 4.4 검증

- 같은 풀에서 **오피스가 격식 있는 아이템을 데일리보다 위로** 올리는지 스냅샷 테스트로 고정.
- mode 보정이 퍼스널컬러·조화의 우선순위를 깨지 않는지(작은 가중인지) 확인.

---

## 5. 한 줄 요약

지금은 "안 먹는 토글"을 제거했다. 다시 넣을 땐 `mode`를 제목이 아니라 **점수 보정**으로 만들고, 격식·무드는 type 키워드 → (이후) 메타데이터/LLM 태깅 순으로 정밀화한다. CSS는 재사용을 위해 남겨 뒀다.
