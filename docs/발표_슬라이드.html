프로젝트 개요

퍼스널컬러 AI 옷장 어시스턴트

얼굴 사진 + 설문 5문항으로 12계절 퍼스널컬러를 진단하고, 그 결과로 내 옷을 색 적합도순 코디로 추천합니다.

왜 만들었나
  • 전문가 대면 진단은 비용(5~15만원)과 접근성의 벽
  • 진단을 받아도 실제 옷 선택과 연결되지 않는 단절
  • 색상과 날씨를 동시에 고려한 코디 추천은 드뭄

개선하게 된 이유

왜 추천 알고리즘인가

이 프로젝트의 심장은 추천입니다. 진단·옷장·날씨는 모두 “그래서 뭘 입을까”를 위한 입력이고, 사용자가 가치를 느끼는 지점도 추천 결과입니다.

그런데 — 동작은 했지만 설명할 수 없었다
  • 왜 이 코디인지 설명 못 하는 블랙박스
  • 점수 계수가 근거 없는 매직 넘버
  • 개별 의류 점수 중심 — 코디 단위 완성도 못 봄
  • 추천 로직이 App.tsx에 섞여 검증·설명 불가

졸업작품에서 가장 방어해야 할 질문은 “그 추천이 왜 타당한가”. → 설명 가능하고 검증된 엔진으로 만드는 것을 최우선으로.

추가 발전하게 된 이유

왜 데일리룩·단일 옷 추천인가

추천이 탄탄해져도 경험은 “리스트를 보는 것”에서 멈춥니다. 진짜 개인화는 한 걸음 더 — 내 상황 맞춤과 결과물 완성까지.

단일 옷 코디 추천

“오늘 이 옷을 입고 싶은데 뭘 매치하지?” — 고정하고 싶은 한 벌에서 출발하는 추천.

데일리룩

마음에 든 코디를 보드에 배치하고 텍스트까지 얹어 “나만의 룩 이미지”로 완성.

경험을 “진단 → 추천”에서 “진단 → 추천 → 내 상황 맞춤 → 결과물 완성”으로 확장.

개선사항 · 추천 UI

블랙박스 추천 → 근거를 화면에

  • 동적 헤드라인 — 시즌·배색·점수가 들어간 한 줄
  • 근거 3줄 — 퍼스널핏·색조화·날씨를 각각 점수와 함께
  • 4축 점수 분해 — 어느 축에서 점수가 났는지 노출
  • 품질 진단 · 출처 뱃지 — 기준 대비 가산/감산, 보유/카탈로그 표시

사용자가 추천을 신뢰할 근거를 화면에서 바로 확인합니다.

추가발전 · 신규 기능

이 옷으로 코디 찾기

  • 옷 한 벌을 기준으로 고정 → 어울리는 코디 탐색
  • 상대 옷 출처 선택: 내 옷장 / 카탈로그 / 둘 다
  • 기존 4축 엔진을 그대로 재사용해 점수 산식 공유
